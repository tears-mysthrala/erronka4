# Web Iragazketa Politika

**Helburua:** Web nabigazio segurua bermatzea eta produktibitatea babestea.

## 1. Iragazketa Kategoriak

### 1.1 Erabat Blokeatuta
Sarbidea blokeatuta dago kategoria hauetan:
- **Malware eta Phishing Guneak:** Mehatxu ezagunen zerrenda.
- **Helduentzako Edukia / Pornografia:** Lan ingurunean desegokia.
- **Joko eta Apustuak:** Produktibitatea kaltetzen duten orriak.
- **P2P eta Legez Kanpoko Deskargak:** Torrent, megaupload antzekoak.
- **Anonymizers / Proxies:** VPN edo proxy guneak politika saltatze aldera.
- **Drogak eta Alkoholismoa:** Eduki arriskutsua.

### 1.2 Alertarekin Baimendua (Monitorizatua)
- **Sare Sozialak:** Facebook, Twitter, LinkedIn, Instagram (monitorizatua).
- **Stream-ing Zerbitzuak:** YouTube, Netflix, Spotify (monitorizatua).
- **Posta Pertsonala:** Gmail, Hotmail, Yahoo (monitorizatua).

### 1.3 Beti Baimendua
- Lan tresnak (Google Drive, Dropbox).
- Eguneroko albisteak (prentsa legitimoa).
- Hornitzaileen webguneak (AWS, Microsoft, GitHub).

## 2. Monitorizazioa

### 2.1 Log Gordetzea
- Nabigazio log-ak gorde daitezke segurtasun intzidenteak ikertzeko.
- Log-ak 90 egunez gordeko dira.
- Log-ak zifraturik gordeko dira eta sarbide mugatua izango dute.

### 2.2 Pribatutasunaren Errespetua
- Ez da monitorizazio indiskriminaturik egingo (pribatutasuna errespetatuz).
- Nabigazio azterketa intzidenteak ikertzeko bakarrik egingo da.
- Log-ak ikusteko CISO edo IT Kudeatzailearen baimena behar da.

### 2.3 GDPR Betetzea
- Langileei jakinarazi behar zaie monitorizazioa egiten dela.
- Datu pertsonalak babestuta egongo dira.

## 3. Salbuespenak

### 3.1 Blokeatutako Gune baten Desblokeoa
- Lanerako beharrezkoa bada blokeatutako gune bat, IT-ri eskatu daiteke desblokeoa:
  - Aldi baterako desblokeoa (24 ordu, 7 egun).
  - Desblokeoa iraunkorrerako (justifikatu behar da).
- Eskaera tiketa bidez egin behar da justifikazioarekin.

### 3.2 Onarpena
- Desblokeo aldi baterakoa: IT teknikariak onar dezake.
- Desblokeo iraunkorra: IT Kudeatzailea edo CISOk onartu behar du.

## 4. Teknologia

### 4.1 Web Filter Sistema
- **Fortinet FortiGate:** Firewall integratua web filter-rekin.
- **DNS Filtering:** DNS mailako blokeaketa.
- **SSL Inspection:** HTTPS trafikoa ikuskatzea (ziurtagirirekin).

### 4.2 Kategoria Eguneratzea
- Kategoria zerrendak egunero eguneratuko dira automatikoki.
- Mehatxu berri bat detektatzen bada, berehala blokeatu behar da.

## 5. Arau-hausteak

### 5.1 Politikaren Urraketa
- **Lehen aldiz:** Ahozko ohartarazpena.
- **Bigarren aldiz:** Idatzizko ohartarazpena espedientean.
- **Hirugarren aldiz edo oso larria:** Diziplina espedientea, kaleratzea.

### 5.2 Urraketa Larriak
- Malware edo phishing guneak saiatzea atzitzea.
- Proxy edo VPN bat erabiltzea politika saltatzeko.
- Pornografia edo helduentzako eduki bat atzitzea.

## 6. Salbuespen Bereziak

### 6.1 Segurtasun Profesionalak
- Segurtasun profesionalek (CISO, Segurtasun Analistak) baimena dute mehatxu guneak ikertzeko.
- Sarbide hau monitorizatuko da eta justifikatu behar da.

### 6.2 Zuzendaritza
- Zuzendaritzak salbuespen bat eskatu dezake LAN beharrezkoa bada.
- Salbuespena dokumentatu eta onartu behar da.

## 7. Prestakuntza eta Kontzientziazioa

### 7.1 Langileen Prestakuntza
- Urteroko prestakuntzan web iragazketa politika azaldu behar da.
- Phishing eta social engineering identifikazioa landu behar da.

### 7.2 Ohartarazpenak
- Blokeatutako gune batera sartu nahi badute, langileei mezu bat agertuko zaie zergatik blokeatuta dagoen azalduz.

## 8. Erantzukizunak
- **IT Sailak:** Web filter sistema kudeatu, kategoriak eguneratu, salbuespena kudeatu.
- **CISO:** Politika definitu, arau-hausteak ikuskatu.
- **Erabiltzaileak:** Politika errespetatu, politika saltatzeko saiakeretan ez parte hartu.

## 9. Berrikuste Plana
- Web iragazketa politika urtero berrikusi.
- Kategoria zerrendak hiruhilero berrikusi.

**Erantzukizuna:** IT Kudeatzailea + CISO
**Berrikuste Maiztasuna:** Urtero
